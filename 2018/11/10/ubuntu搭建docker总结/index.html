ubuntu搭建docker总结

选择系统

使用腾讯云Docker容器：
在Ubuntu Server 12.04 LTS 64位上集成软件:docker 1.8.2

不用再搭建docker，docker daemon将自动运行，而无需配置
docker命令:

docker ps 列出所有运行中的容器
docker ps -a 列出所有容器（含沉睡镜像）
docker images 列出本地所有镜像
docker pull 从 Docker Hub 中拉取或者更新指定镜像

搭建过程

首先docker images,docker ps 都是没有任何东西的

第一步下载镜像：
docker pull chriswayg/apache-php 下载一个apache-php环境的镜像

有其他需要可以下载其他的镜像

创建docker容器
docker run --name group4_80 -p 0.0.0.0:4000:80 -d -it chriswayg/apache-php /bin/bash
  • -p 0.0.0.0:主机端口:虚拟机端口
  • -name 容器名字
  • -d 后台运行容器，并返回容器ID
  • -i: 以交互模式运行容器，通常与 -t 同时使用；
  • -t: 为容器重新分配一个伪输入终端，通常与 -i 同时使用

命令解释：使用镜像chriswayg/apache-php以交互模式启动一个容器,在容器内执行/bin/bash命令。

运行docker容器
docker start group4_80
#docker stop group4_80 //关闭容器
#docker restart group4_80 //重启容器
#docker rm group4_80 //删除容器
#docker pause group4_80 //暂停容器的所有进程
#docker unpause group4_80 //恢复容器的所有进程。
进入docker容器
docker attach group4_80
docker cp 用于容器与主机之间的数据拷贝

实例
将主机/var/www目录拷贝到容器a81b15bb20ab的/var/www/html目录下

docker cp /var/www a81b15bb20ab:/var/www/html

将容器a81b15bb20ab的/www目录拷贝到主机的/tmp目录中。

docker cp  a81b15bb20ab:/www /tmp/

如何查看容器ID
docker ps

后续步骤

当访问ip:4000 时连接被拒绝。原因是没有开启apache
进入docker后开启apache:

/etc/init.d/apache2 start  #restart|stop
#service httpd start

docker容器返回错误：
docker E: Unable to locate package nginx
在使用docker容器时，有时候里边没有安装vim，敲vim命令时提示说：vim: command not found，这个时候就需要安装vim，可是当你敲apt-get install vim命令时，提示：

        Reading package lists... Done
        Building dependency tree       
        Reading state information... Done
        E: Unable to locate package vim
        这时候需要敲：apt-get update

这个命令的作用是：同步 /etc/apt/sources.list 和 /etc/apt/sources.list.d 中列出的源的索引，这样才能获取到最新的软件包。

        等更新完毕以后再敲命令：apt-get install vim命令即可

移动当前目录下所有的文件到上一级目录
mv * ../
强制删除没有权限删除的文件 cb文件夹是一个目录,可用rm -rf cb删除
chmod修改目录权限

chmod [ u / g / o / a ] [ + / - / = ] [ r / w / x ] file
其中 ：
u表示User，是文件的所有者
g表示跟User同Group的用户
o表示Other，即其他用户
a表示ALL，所有用户
-------------------------------------
+表示增加权限
-表示取消权限
=表示取消之前的权限，并给予唯一的权限
-------------------------------------
r表示Read，即读文件
w表示Write，即写文件
x表示运行文件

取消其他用户的写的权限： chmod o-w html
